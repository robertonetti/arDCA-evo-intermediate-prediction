# arDCA-paths

Autoregressive DCA for reconstructing evolutionary intermediate sequences, as described in:

> **On reconstructing evolutionary intermediates using generative sequence models**  
> Roberto Netti and Martin Weigt, *bioRxiv*, 2026.

## Overview

This package trains autoregressive DCA models to learn the conditional distribution
P(S_mid | S_start, S_end) of an intermediate sequence given the two evolutionary endpoints.
It is designed to work with trajectory triplets (S_start, S_mid, S_end) generated by in silico evolution
in a bmDCA sequence landscape (see [Di Bari et al., PNAS 2024](https://doi.org/10.1073/pnas.2406807121)).

The trained model can then reconstruct evolutionary intermediates either by:
- **greedy maximum-likelihood decoding** — maximizes residue-level accuracy but produces statistically atypical sequences;
- **autoregressive sampling** — reproduces the correct distribution of intermediates, including their statistical energy and Hamming distances from the endpoints.

## Installation

The package requires `adabmDCA` and its dependencies.

```bash
git clone https://github.com/robertonetti/arDCA-evo-intermediate-prediction.git
cd arDCA-evo-intermediate-prediction
python -m pip install .
```

## Input data format

Training data must be provided as a FASTA file where each sequence is the concatenation of a triplet:

```
S_start | S_end | S_mid      (for --mode third, default)
S_start | S_end              (for --mode second, start+end only)
```

Each timescale τ ∈ {10¹, 10², 10³, 10⁴, 10⁵, 10⁶} MC steps requires a separate model trained on the corresponding triplet dataset (70k train / 10k validation / 20k test).

## Training

All training options are listed with:

```bash
python -m arDCA_paths.scripts.train -h
```

### Minimal training command

```bash
python -m arDCA_paths.scripts.train \
    -d <path_to_train_fasta> \
    --data_test <path_to_test_fasta> \
    -o <output_folder> \
    -l <label> \
    --mode third
```

### `--mode` options

| Mode | Description |
|------|-------------|
| `third` (default) | Predicts S_mid conditioned on S_start and S_end. Input FASTA contains sequences of length 3L, organized as (S_start ‖ S_end ‖ S_mid). This is the main mode used in the paper. |
| `second` | Like `third`, but internally discards the middle third of the concatenated sequence and uses only the first and last thirds as conditioning. Intended for alternative conditioning experiments. |
| *(unset)* | Standard arDCA training with no path conditioning. Equivalent to the original arDCA model. |

### Key arguments

| Argument | Default | Description |
|----------|---------|-------------|
| `-d` / `--data` | — | Training FASTA file (required) |
| `--data_test` | None | Validation/test FASTA file |
| `-o` / `--output` | `DCA_model` | Output folder |
| `-l` / `--label` | None | Label prefix for output files |
| `--mode` | `third` | Training mode (see above) |
| `--lr` | 0.005 | Learning rate |
| `--reg_h` | 1e-6 | L2 regularization on fields |
| `--reg_J` | 1e-4 | L2 regularization on couplings |
| `--nepochs` | 1000 | Maximum number of epochs |
| `--epsconv` | 5e-2 | Convergence threshold on the loss |
| `--batch_size` | None | Mini-batch size (full-data mode if None) |
| `--no_entropic_order` | False | Disable entropic ordering of sites |
| `--path_graph` | None | Path to a pre-computed interaction graph |
| `--device` | `cuda` | Computation device (`cuda` or `cpu`) |
| `--seed` | 0 | Random seed |

### Output files

After training, the following files are saved in `<output_folder>/`:

| File | Description |
|------|-------------|
| `<label>_params.pth` | Parameters of the last epoch |
| `<label>_params_best.pth` | Parameters of the epoch with lowest validation loss |
| `<label>_weights.dat` | Sequence reweighting coefficients (if not provided) |
| `loss.png` | Training and validation loss curves |
| `pearson.png` | Pearson correlation of two-site statistics during training |

> **Tip:** For downstream predictions, use `params_best.pth`. The last-epoch model is saved for compatibility but is generally not the best checkpoint.

## Typical workflow (paper experiments)

```bash
# 1. Train one model per timescale
for TAU in 1e1 1e2 1e3 1e4 1e5 1e6; do
    python -m arDCA_paths.scripts.train \
        -d data/chorismate/train_tau${TAU}.fasta \
        --data_test data/chorismate/test_tau${TAU}.fasta \
        -o models/chorismate/tau${TAU} \
        -l tau${TAU} \
        --mode third \
        --batch_size 2048 \
        --nepochs 500
done

# 2. Sample intermediates from a trained model (greedy or stochastic)
python -m arDCA_paths.scripts.sample \
    -p models/chorismate/tau1e4/tau1e4_params_best.pth \
    -d data/chorismate/test_tau1e4.fasta \
    -o results/tau1e4 \
    --ngen 20000
```
